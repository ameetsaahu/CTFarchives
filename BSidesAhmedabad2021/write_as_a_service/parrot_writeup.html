Getting libc leak in write as a service.

first stdout overwrite:

both printf and fputs use _IO_new_file_xsputn(https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L1197) to write stuff. At some point it can call _IO_OVERFLOW(https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L1244), this part of _IO_overflow is useful for our situation ( https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L766). we can set write_base,ptr,end to read_base. here we will partially overwrite two last bytes ( require 4bit bruteforce ) of read_base to somewhere above write_base ( will be exaplained later ). to reach there, we just have to unset _IO_CURRENTLY_PUTTING flag. note that we have to set _IO_LINE_BUF to overwrite write_end too (https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L773). It also sets _IO_CURRENTLY_PUTTING flag too which is useful later (https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L771 ). finally new_do_write(https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L431) will be called, the end of the function looks scary since it will set io_write,io_base pointers to io_buf_base, but there is sysseek at there https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L443, and it always fails (Illegal seek). so as long as we don't set _IO_IS_APPENDING we are fine. so after overwriting flag and two last byte of read_base, printf inside menu will call _IO_new_file_xsputn and above stuff will happen. Then we will have following stdout.

0x00001555555186a0│+0x0000: 0x00000000fbad2a85

You can see that write_ptr is pointing to read_end. After sending 1 for our second overwrite, content?\n> will be passed to _IO_new_file_xsputn. Since _IO_OVERFLOW has set _IO_CURRENTLY_PUTTING and _IO_LINE_BUF is also set, xsputn will copy content?\n to write_ptr(https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L1236), this will overwrite last byte of write_base with newline.

second stdout overwrite:

If we don't stop buffering, printf inside menu will corrupt stdout. to stop buffering, we can pick _IO_LINE_BUF flag, and then buffering will be disabled xd since then we have not satisfied both conditions inside xsputn for buffering (https://elixir.bootlin.com/glibc/glibc-2.31/source/libio/fileops.c#L1211). we also set _IO_IS_APPENDING to skip that sysseek and enable writing again. then we will have leaks!
